Prerequisite Skills

You should adept with Subversion and revision control practices.  You may happier with TortoiseSvn or something built into your Intedgrated Development Environment.  

It is most likely that you are a software developer, or web design professional.  At the very least you're a 'end user' who has been given a crash course in basic Subversion usage.

You might anyway want to reaquaint yourself via a Subversion cheetsheet or buy a good book on subversion.

Getting the page content offline.

Do a checkout of the source using normal svn tools.  For Cozmos' own site that would be:

  svn co https://svn.codehaus.org/cozmos/cozmosSiteContent/

You should not need a user account and password to checkout, but you may well need one to push back.

Then choose your tool for editing (see below).

After you've made you changes you can push the whole set of changes back to the subversion server (and the live Cozmos site implicitly) with a subversion commit.

  svn commit -m "a suitable description for the changes"

Reverting changes.

You mean one of two things here :-

Abandoning a change before you commit it:

  svn revert FileName.html

Undoing a change that has already been made:

  svn merge -r 124:121 FileName.html
  svn commit FileName.html -m "ooops - undo new press release as not ready to go live yet"

The two numbers are the latest revision and the one your trying to go back to.
