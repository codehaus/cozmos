Prerequisite skills and server setup

You should be able to install the Java Development Kit and Java web containers as well as modestly configure mod_dav_svn.

Its best to use a version of Linux that has Apache pre-installed and mod_sav_svn installed giving a working https capability for regular Subversion.

Install the Cozmos web app into Apache's Tomcat server

Alpha verions of Cozmos available here here (for the standalone version of Cozmos) and here for the version that stores content in Subversion.  For the latter, you'll need to pre-populate your Subversion content root with some content. All of these archives are pretty small.

If you are using the standalone version and want to serve using Mortbay's Jetty web server, it will work as well as Tomcat, but you must  have JETTY_HOME as an enviromental variable set in order for the content to be stored after the web server is bought down.

If you are using the Subversion depending version, you need to work on the web.xml in WEB-INF inside the WAR file.  There's a URL there (that must end in a slash, but is marked with "TODO" as shipped) that points to the mod_dav_svn (https://) entry point that you're planning to store content in.  You will need to push content into there before you launch the web container.  You will also need to have turned autorevisioning on for mod_dav_svn.
