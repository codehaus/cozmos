Viewing Cozmos pages

Use any Web Browser on any Operating system. 

Editing Pages in a Cozmos system using SeaMonkey

For now (early 2007) you can only* use an application called SeaMonkey from the Mozilla Foundation (the same people that make Firefox).  And even having limited editing to one application, you have to modify it slightly after installation.

1) Go get SeaMonkey from http://www.mozilla.org/projects/seamonkey/ (any recent version).

2) Install it the normal way.

3) Go to to a directory called 'chrome' that contains a file called 'comm.jar' in a command shell

4) Do jar -xf comm.jar content/editor/editorApplicationOverlay.js

5) Edit that file

6) Change ....
  function editPage(url, launchWindow, delay)
  {
  // Always strip off "view-source:" and #anchors
  url = url.replace(/^view-source:/, "").replace(/#.*/, "");
  to ....
  function editPage(url, launchWindow, delay)
  {
  // Always strip off "view-source:" and #anchors
  url = url.replace(/^view-source:/, "").replace(/#.*/, "");
  if (url.substr(0,4) == "http")
  {
    if (url.indexOf("?") > 0)
    {
      url = url + "&editMode=true";   
    }
    else
    {
      url = url + "?editMode=true";       
    }
  }
7) Do jar -uf comm.jar content/editor/editorApplicationOverlay.js

Now your SeaMonkey has been modified. It allows the server side Cozmos to recognise the differences betweeen a page request from the browser and one from the composer (editor) sides of SeaMonkey.  We hope to see the SeaMonkey team build in such functionality in future.

It is important to note that there are no permissions presently for editing of pages.  This will change in future versions of Cozmos.

We will also supply a script to modify SeaMonkey for you in future versions.

8) On any page you want to edit, while in SeaMonkey, choose 'Edit' from the 'File' menu. Alternatively, press Ctrl-E or Apple-E (Windows/Linux vs Mac).  When your finished editing, hit the publishbutton in the toolbar to push the changes back.

Editing Pages in a Cozmos system using other browsers

In time we hope that Firefox gets a full and pluggable HTML editing component, and that Internet Explorer gets a clean one (Word is not quite good enough).  For Safari (Mac) Konqueror (Linux) and others changes will similarly need to be made to be fully editable-with-WebDAV compatible.  As of early 2007, none of these browsers are compatible.
